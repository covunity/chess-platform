# QA Report Template

Use this template when posting the QA result as a GitHub issue comment.

```markdown
> *This was generated by an AI QA agent.*

## QA Report — Issue #<number>

**Verdict:** ✅ Pass / ❌ Fail / ⚠️ Partial

**Commits reviewed:** `<hash1>`, `<hash2>`
**Tests run:** `<command>` — <N> passed, <M> failed / no test suite found

---

### Acceptance criteria

| # | Criterion | Status | Evidence |
|---|-----------|--------|----------|
| 1 | Short description of criterion | ✅ Pass | e.g. `ClassName.method()` returns correct value per code at `src/foo.ts` |
| 2 | Short description of criterion | ❌ Fail | e.g. Error case not handled — `null` input throws unhandled exception |
| 3 | Short description of criterion | ⚠️ Partial | e.g. Logic correct but no test covers this path |

---

### Regressions

- None detected. / List any pre-existing tests that now fail.

---

### Required fixes  *(only present when verdict is Fail or Partial)*

- [ ] Specific fix 1 — describe exactly what code needs to change and why
- [ ] Specific fix 2

---

### Notes  *(optional)*

Any observations that don't affect the verdict but are worth flagging
(e.g. missing test coverage, inconsistent naming, unhandled edge case
that is out of scope for this issue).
```

## Filling in the template

**Verdict rules:**
- ✅ Pass — every criterion met, no regressions found.
- ❌ Fail — at least one criterion not met, or a regression introduced.
- ⚠️ Partial — all criteria technically met but with meaningful caveats
  (no tests, unverifiable external dependency, narrow edge case unhandled).

**Evidence column rules:**
- Quote function names, types, or config keys — concrete handles the next
  agent can grep for.
- Never reference file paths or line numbers (they go stale).
- For ❌ Fail rows, the evidence is the specific gap, not a vague "missing".
- For ⚠️ Partial rows, state what would be needed to fully verify.

**Required fixes section:**
- Only include when verdict is ❌ Fail or ⚠️ Partial.
- Each item must be independently actionable by the coding agent.
- Match the language of the project's domain glossary (from `CONTEXT.md`).
