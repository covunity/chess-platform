# Supabase CLI — Command Reference

## Installation

```bash
npm install -g supabase
supabase --version
```

---

## Authentication & Project Linking

```bash
# Log in (opens browser to retrieve token)
supabase login

# Link local project to a Supabase cloud project
supabase link --project-ref <project-ref>
# project-ref = the subdomain in your URL: https://<project-ref>.supabase.co
# Will prompt for DB password (Settings → Database → Database password)
```

---

## Migrations

```bash
# Create a new migration (generates an empty file in supabase/migrations/)
supabase migration new <migration_name>
# Example: supabase migration new add_orders_table

# List all migrations and their status (applied / pending)
supabase migration list

# Push all pending migrations to Supabase cloud
supabase db push

# Pull schema from cloud and generate a local migration file
supabase db pull
```

> **Migration file naming convention:** `001_<name>.sql`, `002_<name>.sql` — always increment; never reorder files after they have been pushed.

## Local Development (Docker)

```bash
# Start local Supabase stack (PostgreSQL + Auth + Studio running via Docker)
supabase start

# Stop local stack
supabase stop

# Print URLs and keys for the local instance
supabase status
# → API URL: http://127.0.0.1:54321
# → anon key: eyJ...
# → Studio:   http://127.0.0.1:54323

# Reset local DB to a clean state (reruns all migrations + seed)
supabase db reset
```

## Generate TypeScript Types

Run after every schema change to keep types in sync with the database.

```bash
# From Supabase cloud
supabase gen types typescript --project-id <project-ref> \
  --schema public > src/types/database.ts

# From local instance
supabase gen types typescript --local \
  --schema public > src/types/database.ts
```

Usage in code:
```typescript
import type { Database } from '@/types/database'
const supabase = createClient<Database>(url, key)
```

## Edge Functions

```bash
# Create a new Edge Function
supabase functions new <function_name>
# Creates: supabase/functions/<function_name>/index.ts

# Serve function locally (hot-reload)
supabase functions serve <function_name>

# Deploy a single function to cloud
supabase functions deploy <function_name>

# Deploy all functions
supabase functions deploy

# Stream function logs
supabase functions logs <function_name>
```

## Storage

```bash
# List buckets
supabase storage ls

# Upload a file to a bucket
supabase storage cp ./local-file.png ss:///bucket-name/path/file.png

# Download a file from a bucket
supabase storage cp ss:///bucket-name/path/file.png ./local-file.png
```

---

## Secrets (Environment Variables for Edge Functions)

```bash
# Set a secret
supabase secrets set MY_SECRET=value

# Set multiple secrets from a .env file
supabase secrets set --env-file .env.production

# List secrets (values are hidden)
supabase secrets list
```

## Inspect & Debug DB

```bash
# Open a psql session connected to the cloud DB
supabase db connect

# Performance diagnostics
supabase inspect db slow-queries
supabase inspect db unused-indexes
supabase inspect db index-sizes

# Table bloat
supabase inspect db table-sizes
```

## Common Commands for This Project

| Situation | Command |
|-----------|---------|
| Add a new table | `supabase migration new <name>` → write SQL → `supabase db push` |
| Sync types after schema change | `supabase gen types typescript --project-id <ref> > src/types/database.ts` |
| Deploy a new Edge Function | `supabase functions deploy <name>` |
| Check migration status | `supabase migration list` |
| Debug DB locally | `supabase start` → open Studio at `http://127.0.0.1:54323` |

## Required File Structure

```
supabase/
├── config.toml          # local dev config (generated by supabase init)
├── migrations/          # SQL migration files in sequential order
│   ├── 001_users_table.sql
│   ├── 002_admin_update_policy.sql
│   └── 003_courses_chapters_lessons.sql
└── functions/           # Edge Functions (Deno / TypeScript)
    └── <function_name>/
        └── index.ts
```

> Commit the entire `supabase/` directory to git — migrations are the source of truth for the database schema.
